# Configuration

Configuration files and credential storage locations.

## Configuration directory

All configuration is stored in:

```
~/.config/formseal-fetch/
```

| File | Contents | Format |
|------|----------|--------|
| `config.json` | Provider, output folder | Plain JSON |
| `secrets.json` | API token, namespace ID | Base64-encoded JSON (fallback only) |

## config.json

Created when you run `fsf connect`. Contains non-sensitive configuration:

```json
{
  "provider": "cloudflare",
  "output_folder": "data"
}
```

**Fields:**

| Field | Type | Description |
|-------|------|-------------|
| `provider` | string | Storage backend name |
| `output_folder` | string | Path to save ciphertexts |

Provider-specific non-sensitive fields (e.g. `key_prefix` for Redis) are also stored here.

Edit `config.json` directly or reconnect with different values:

```bash
fsf connect <name>
```

## secrets.json

**Only exists** if OS keychain is unavailable and fallback was triggered. Contains base64-encoded credentials:

```json
{
  "cloudflare:token": "Y2Z1dC4uLg==",
  "cloudflare:namespace": "MGYyYjJkY2YuLi4="
}
```

This file is not encrypted — only base64 encoded. It exists as a last-resort fallback.

## OS keychain

Credentials stored in OS keychain are encrypted by the operating system:

- **Windows**: Credential Manager
- **macOS**: Keychain
- **Linux**: DBus Secret Service

formseal-fetch manages credentials automatically. Use `fsf status` to verify the storage location.

## Runtime behavior

1. On `fsf connect`: Credentials saved to OS keychain (preferred) or secrets.json (fallback)
2. On `fsf fetch`: Credentials loaded from OS keychain first, then secrets.json
3. On `fsf disconnect`: Credentials deleted from both

## Removing configuration

```bash
fsf disconnect
```

Removes config.json, secrets.json (if exists), and credentials from OS keychain. Downloaded ciphertexts are not affected.
