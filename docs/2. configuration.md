# Configuration

Configuration files and credential storage locations.

## Configuration directory

All configuration is stored in:

```
~/.config/formseal-fetch/
```

| File | Contents | Format |
|------|----------|--------|
| `config.json` | Provider, output folder | Plain JSON |

## config.json

Created when you run `fsf connect`. Contains non-sensitive configuration:

```json
{
  "provider": "cloudflare",
  "output_folder": "data"
}
```

**Fields:**

| Field | Type | Description |
|-------|------|-------------|
| `provider` | string | Storage backend name |
| `output_folder` | string | Path to save ciphertexts |

Provider-specific non-sensitive fields (e.g. `key_prefix` for Redis) are also stored here.

Edit `config.json` directly or reconnect with different values:

```bash
fsf connect <name>
```

## OS keychain

Credentials (API tokens, namespace IDs) are stored in your OS keychain:

| OS | Storage |
|---|---------|
| Windows | Credential Manager |
| macOS | Keychain |
| Linux | DBus Secret Service |

formseal-fetch manages credentials automatically. Use `fsf status` to verify the storage location.

## Runtime behavior

1. On `fsf connect`: Credentials saved to OS keychain
2. On `fsf fetch`: Credentials loaded from OS keychain
3. On `fsf disconnect`: Credentials deleted from OS keychain

## Removing configuration

```bash
fsf disconnect
```

Removes config.json and credentials from OS keychain. Downloaded ciphertexts are not affected.
