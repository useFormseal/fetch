# How it works

## The flow

```
formseal-embed (browser encryption)
       |
       v (encrypted submissions)
  Storage backend
       |
       v (fsf fetch)
  formseal.ct.jsonl
       |
       v ([formseal-decrypt](https://github.com/useFormseal/decrypt))
  formseal.decrypted.jsonl
       |
       v
  You
```

1. [formseal-embed](https://github.com/useFormseal/embed) encrypts form submissions in the browser
2. Ciphertexts are stored at your backend (prefixed `formseal.`)
3. `fsf fetch` downloads them to `formseal.ct.jsonl`
4. [formseal-decrypt](https://github.com/useFormseal/decrypt) decrypts them locally

## What fsf does

- Connects to supported storage backends (Cloudflare KV, Supabase, Redis)
- Downloads ciphertexts to a JSONL file
- Skips duplicates automatically on repeated runs
- Stores credentials in OS keychain
- Never sends data to any server except your storage backend

## What fsf does NOT do

- fsf does not decrypt data — you decrypt offline with formseal-decrypt
- Does not send data to external servers
- Does not store plaintext anywhere
- Does not require your private key

## Position in the ecosystem

| Step | Tool | What happens |
|------|------|-------------|
| 1 | [formseal-embed](https://github.com/useFormseal/embed) | Encrypts submission in the browser |
| 2 | formseal-fetch | Downloads ciphertexts from storage (this tool) |
| 3 | [formseal-decrypt](https://github.com/useFormseal/decrypt) | Decrypts locally |

## Decryption

To decrypt ciphertexts locally:

```bash
pip install formseal-decrypt
fsd connect
fsd decrypt
```

For full formseal-decrypt documentation, see the [formseal-decrypt repo](https://github.com/useFormseal/decrypt).
