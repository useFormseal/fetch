# Backends

Connecting formseal-fetch to your storage backend.

## Cloudflare KV

### What you need

| Item | Where to find it |
|------|------------------|
| KV Namespace ID | Cloudflare Dashboard → Workers → KV |
| API Token | Cloudflare Dashboard → Profile → API Tokens |

### Getting your credentials

**KV Namespace ID:** Workers → KV → copy the **ID** column.

**API Token** must have these permissions:

| Permission | Required for |
|------------|---------------|
| Account Settings: Read | Listing your account |
| Workers KV: Read | Reading from KV namespace |

### Connecting

```bash
# Interactive
fsf connect cloudflare

# Non-interactive
fsf connect cloudflare namespace:<id> token:<value> output:<path>
```

### Status output

```
Provider:                 ⛅️ Cloudflare
Account ID:               1d4ed052c5e97efc*****
KV Namespace ID:          0f2b2dcf9dd94e4285d476043af3c26d
Token Location:           OS Keychain
```

### Common issues

| Error | Cause | Solution |
|-------|-------|----------|
| HTTP 401 | Invalid or expired API token | Generate a new token in Cloudflare Dashboard |
| "No accounts found" | Token lacks `Account Settings: Read` permission | Update token permissions in Cloudflare Dashboard |
| Empty results | KV namespace has no keys or wrong namespace ID | Verify namespace ID in Cloudflare Dashboard |

## Supabase

### What you need

| Item | Where to find it |
|------|------------------|
| Project Reference | Your Supabase project URL (`[ref].supabase.co`) |
| Table Name | Table storing ciphertexts |
| Service Role Key | Supabase Dashboard → Settings → API |

### Getting your credentials

**Project Reference:** From your project URL — use just the `[ref]` part.

**Table Name:** The table where ciphertexts are stored (e.g. `submissions`, `ciphertexts`).

**Service Role Key:** Settings → API → copy `service_role` key (not `anon` key). The service_role key bypasses RLS.

### Connecting

```bash
# Interactive
fsf connect supabase

# Non-interactive
fsf connect supabase project_ref:<ref> table:<name> token:<key> output:<path>
```

### Common issues

| Error | Cause | Solution |
|-------|-------|----------|
| HTTP 403 | Using anon key instead of service_role | Use service_role key |
| HTTP 404 | Invalid project reference | Check project URL |
| "No table" | Wrong table name | Verify table exists |
| Empty results | Table has no rows or wrong columns | Check table data |

## Redis

### What you need

| Item | Where to find it |
|------|------------------|
| Redis URL | Connection string (e.g. `rediss://user:pass@host:port`) |
| Key prefix | List key where ciphertexts are stored |

### Getting your credentials

**Redis URL:** For Upstash Redis — copy from Connection Details. For self-hosted — `redis://host:port` or `rediss://host:port` for TLS.

**Key prefix:** The Redis list key. Default is `submissions`.

### Connecting

```bash
# Interactive
fsf connect redis

# Non-interactive
fsf connect redis key_prefix:submissions output:data
```

### Common issues

| Error | Cause | Solution |
|-------|-------|----------|
| "Unable to connect to Redis" | Invalid URL or unreachable | Check Redis URL and network |
| "Not set" | Token not saved | Reconnect with correct URL |
| Empty results | Wrong key prefix | Check prefix matches (case-sensitive) |
