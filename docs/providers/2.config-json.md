# 1. config.json

Defines provider metadata and user input fields.

---

## Create the folder

```
fsf/providers/<name>/
```

---

## Create config.json

```json
{
  "display_name": "Display Name",
  "storage_type": "PostgreSQL",
  "inputs": [
    {
      "name": "project_ref",
      "description": "Project Reference",
      "required": true,
      "sensitive": false
    },
    {
      "name": "token",
      "description": "API Token",
      "required": true,
      "sensitive": true
    }
  ]
}
```

---

## Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `display_name` | string | Yes | Name shown in `fsf providers` |
| `storage_type` | string | Yes | Backend type (PostgreSQL, KV, etc.) |
| `inputs` | array | No | List of user input fields |

---

## Input field object

Each field in `inputs`:

| Property | Type | Description |
|----------|------|-------------|
| `name` | string | Variable name (passed to engine) |
| `description` | string | Prompt message in `fsf connect` |
| `required` | boolean | Must user provide it? |
| `sensitive` | boolean | Store in OS keychain? If true, hidden in `fsf status` |

When `sensitive` is `true`, the value is stored in the OS keychain and loaded automatically on fetch. Non-sensitive values are stored in `config.json` as plaintext. On `fsf disconnect`, all sensitive fields are wiped from the keychain.

---

## Example: supabase

```json
{
  "display_name": "⚡️ Supabase",
  "storage_type": "PostgreSQL",
  "inputs": [
    {
      "name": "project_ref",
      "description": "Project Reference",
      "required": true,
      "sensitive": false
    },
    {
      "name": "table",
      "description": "Table name",
      "required": true,
      "sensitive": false
    },
    {
      "name": "token",
      "description": "Service Role Key",
      "required": true,
      "sensitive": true
    }
  ]
}
```

Sensitive fields (`token`) are stored in the OS keychain; non-sensitive fields (`project_ref`, `table`) in `config.json`.
