# 1. config.json

Defines provider metadata and user input fields.

---

## Create the folder

```
fsf/providers/<name>/
```

---

## Create config.json

```json
{
  "display_name": "Display Name",
  "storage_type": "PostgreSQL",
  "token_label": "API Token",
  "inputs": [
    {
      "name": "project_ref",
      "description": "Project Reference",
      "required": true,
      "sensitive": false
    }
  ]
}
```

---

## Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `display_name` | string | Yes | Name shown in `fsf providers` |
| `storage_type` | string | Yes | Backend type (PostgreSQL, KV, etc.) |
| `token_label` | string | No | Prompt for API token (default: "Token") |
| `inputs` | array | No | List of user input fields |

---

## Input field object

Each field in `inputs`:

| Property | Type | Description |
|----------|------|-------------|
| `name` | string | Variable name (passed to engine) |
| `description` | string | Prompt message in `fsf connect` |
| `required` | boolean | Must user provide it? |
| `sensitive` | boolean | Hide in `fsf status`? |

---

## Example: supabase

```json
{
  "display_name": "⚡️ Supabase",
  "storage_type": "PostgreSQL",
  "token_label": "Service Role Key",
  "inputs": [
    {
      "name": "project_ref",
      "description": "Project Reference",
      "required": true,
      "sensitive": false
    },
    {
      "name": "table",
      "description": "Table name",
      "required": true,
      "sensitive": false
    }
  ]
}
```
