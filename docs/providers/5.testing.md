# 4. Testing

Verify your provider works correctly.

---

## Test commands

Run these in order:

```bash
# 1. Check provider loads
fsf providers

# 2. Connect (walk through setup)
fsf connect <name>

# 3. Check status
fsf status

# 4. Fetch data
fsf fetch

# 5. Debug mode (if errors)
fsf fetch --debug
```

---

## Expected output: `fsf providers`

```
Available providers:

  >  ⚡️ Supabase:     PostgreSQL
  >  <Name>:         Storage Type
```

---

## Verify siscovery

Your provider should appear automatically after adding:

1. `fsf/providers/<name>/__init__.py`
2. `fsf/providers/<name>/config.json`
3. `fsf/providers/<name>/engine.py`

If not, check for import errors:

```bash
python -c "from fsf.providers import get_providers; print(list(get_providers().keys()))"
```

---

## Debug mode

Use `--debug` for detailed errors:

```bash
fsf fetch --debug
```

This passes `debug=True` to your engine.py `run(config, debug=...)` function.
