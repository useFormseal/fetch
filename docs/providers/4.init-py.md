# 3. __init__.py

Defines the provider class that connects fetch logic to the CLI.

---

## Create __init__.py

```python
# fsf/providers/<name>/__init__.py
# <name> provider

from fsf.providers import BaseProvider
from fsf.providers.<name>.engine import run


class <Name>Provider(BaseProvider):
    name = "<name>"

    def _do_fetch(self, config):
        return run(config, debug=config.get("debug", False))


Provider = <Name>Provider
```

---

## Template breakdown

```python
from fsf.providers import BaseProvider       # Base class
from fsf.providers.<name>.engine import run  # Your engine

class <Name>Provider(BaseProvider):       # Inherit BaseProvider
    name = "<name>"                    # Your unique name

    def _do_fetch(self, config):          # Required method
        return run(config, debug=config.get("debug", False))

Provider = <Name>Provider              # Export this name
```

---

## Key points

1. **Must** inherit `BaseProvider`
2. **Must** define `name` class attribute
3. **Must** implement `_do_fetch(self, config)`
4. **Must** export as `Provider`

---

## Note on `debug`

Pass `debug=config.get("debug", False)` to your engine - this enables `--debug` flag in `fsf fetch`.
