# Getting Started

Welcome to the provider guide! This walks you through creating a new storage provider.

---

## What is a provider?

A provider is a storage backend that `fsf fetch` retrieves ciphertexts from.

Examples:
- **Cloudflare KV** - Key-value store
- **Supabase** - PostgreSQL database
- **Redis** - In-memory data store

---

## Concepts

```
fsf fetch → Provider.fetch() → {"id": ciphertext_data}
```

The provider:
1. Connects to your storage backend
2. Fetches ciphertext records
3. Returns them as `dict[str, bytes]`

---

## Folder structure

Each provider is a folder under `fsf/providers/<name>/`:

```
fsf/providers/
├── __init__.py           # Registry (don't touch)
├── cloudflare/          # Provider folder
│   ├── config.json
│   ├── __init__.py
│   └── engine.py
└── <yourprovider>/    # Your new provider
    ├── config.json
    ├── __init__.py
    └── engine.py
```

---

## Steps overview

| Step | File | What to do |
|------|------|------------|
| 1 | `config.json` | Define metadata + inputs |
| 2 | `engine.py` | Write fetch logic |
| 3 | `__init__.py` | Define provider class |
| 4 | Test | Verify it works |
