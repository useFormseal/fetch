# Commands reference

Complete reference for all formseal-fetch commands.

## Usage syntax

```bash
fsf <command> [options] [arguments]
fsf --<flag>
```

## Commands

### connect

Connect to a storage backend.

```bash
fsf connect <name> [field:value]...
```

**Arguments:**

| Argument | Description |
|----------|-------------|
| `<name>` | Storage provider — `cloudflare`, `supabase`, `redis` |
| `field:<value>` | Provider-specific fields (see [Backends](./3.%20backends.md)) |

**Examples:**

```bash
# Interactive — prompted for all required values
fsf connect cloudflare

# Non-interactive — all values provided inline
fsf connect cloudflare namespace:<id> token:<value>
```

Press `Ctrl+C` at any prompt to cancel. Already configured? Run `fsf disconnect` first — `fsf connect` will refuse.

### fetch

Download ciphertexts from your connected backend.

```bash
fsf fetch [--output <path>] [--debug]
```

**Options:**

| Option | Description |
|--------|-------------|
| `--output` | Custom output file path (default: `<output_folder>/formseal.ct.jsonl`) |
| `--debug` | Show full error tracebacks |

**Examples:**

```bash
# Default output folder
fsf fetch

# Custom output file
fsf fetch --output my-data.jsonl

# Debug mode
fsf fetch --debug
```

**Deduplication:** Running `fsf fetch` multiple times skips duplicates based on ciphertext content. Output shows new entries, duplicates skipped, and current total.

### status

Show current connection status and configuration.

```bash
fsf status
```

**Output includes:**
- Provider name and display icon
- Account ID (truncated, Cloudflare only)
- Provider-specific fields
- Token status and storage location
- Storage type
- Output folder path

### disconnect

Clear all credentials and configuration.

```bash
fsf disconnect
fsf disconnect --wipe
```

**What it removes:**
- Provider configuration
- API token (from OS Keychain or secrets.json)
- Provider-specific credentials
- Configuration file

**What it does NOT remove (disconnect only):**
- Downloaded ciphertexts in your output folder

**--wipe flag:** Also deletes the ciphertexts file.

### --providers

List available storage backends.

```bash
fsf --providers
```

### --version

Show version number.

```bash
fsf --version
```

### --aliases

Show shorthand aliases.

```bash
fsf --aliases
```

| Short | Canonical |
|-------|-----------|
| `-s` | `status` |
| `-c` | `connect` |
| `-p` | `--providers` |

### --help

Show help information.

```bash
fsf --help
```

### No-argument

Show project information.

```bash
fsf
```

## Error behavior

formseal-fetch has two fatal error tiers:

| Tier | Prefix | Exits |
|------|--------|-------|
| `neutral()` | 😐 | Yes |
| `fail()` | `Error:` | Yes |

Both stop execution immediately. `neutral()` is used for user-facing mistakes, `fail()` for system-level issues.

## Examples

### Full workflow

```bash
# Configure
fsf connect cloudflare namespace:<id> token:<value>

# Fetch ciphertexts
fsf fetch

# Check status
fsf status

# Clear credentials when done
fsf disconnect
```
