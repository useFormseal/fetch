# Troubleshooting

## Installation

### "Command not found: fsf"

Package not installed or PATH not updated.

```bash
pip show formseal-fetch
```

## Connection

### "No provider configured. Run: fsf connect <name>"

Never connected or configuration deleted.

```bash
fsf connect <name>
```

### "No token"

Connected but no API token stored.

```bash
# Reconnect to store a new token
fsf connect <name> token:<value>
```

### "Invalid format: ... Use key:value"

Incorrect argument syntax. Use colon to separate key and value:

```bash
# Correct
fsf connect cloudflare namespace:<id>

# Incorrect
fsf connect cloudflare namespace=<id>
```

## Authentication

### "Unable to connect to Redis"

Invalid Redis URL or unreachable host. Possible causes:

- VPN blocking the Redis port
- Firewall restricting the connection
- Network connectivity issues

Run `fsf fetch --debug` for full traceback details.

### "Auth failed: HTTP 401"

API token is invalid or expired.

1. Verify token is correct (no extra spaces)
2. Create a new token in your provider dashboard
3. Reconnect:

```bash
fsf disconnect
fsf connect <name> token:<value>
```

### "No accounts found. Token needs 'Account Settings: Read' scope"

Token lacks required permissions (Cloudflare).

1. Go to Cloudflare Dashboard → Profile → API Tokens
2. Ensure token has Account: Read and Workers KV: Read

## Fetch

### "HTTP 403"

Token doesn't have read permissions on the storage backend.

**Cloudflare:** Add Workers KV: Read permission.  
**Supabase:** Use service_role key, not anon key.

### Empty results

KV namespace has no keys, or wrong key prefix (Redis) or table name (Supabase).

## Credential storage

### "Token Location: Config File"

OS keychain unavailable; fell back to JSON file. This is not an error — credentials still work. Happens when running in a container without keyring support, or keyring service not available.

### Credentials not persisting after reboot

Likely using fallback JSON storage and file was deleted. Reconnect to restore credentials to keychain.

## General

### Ctrl+C during prompt

Cancels gracefully — no changes made. You can safely interrupt at any prompt.

### Still stuck?

```bash
# Debug mode for detailed errors
fsf fetch --debug
```

Check [GitHub Issues](https://github.com/useFormseal/fetch/issues) or open a new issue with the command you ran, full error message, and OS info.
